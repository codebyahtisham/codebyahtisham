# Welcome to My GitHub!

Hi, I'm **Ahtisham Saleem**, an Electrical Engineer and AI Research Engineer building intelligent systems at the intersection of hardware and machine learning.

## What I Do

I currently work as an **AI Research Engineer at E-Intel, LCE (LUMS)**, a deep-tech startup focused on **Brain-Computer Interface (BCI)** systems. My day-to-day involves designing ML pipelines for biosignal classification (EEG/EMG), writing Python for hardware–software integration, and automating everything from data collection to model validation. I'm driven by the idea that AI shouldn't just live in the cloud — it should work on real devices, processing real signals, in real time.

As a final-year **BS Electrical Engineering** student at **Namal University, Mianwali**, I've built a strong foundation across digital systems, embedded design, control theory, and wireless communication. My final year project — a **Cloud-Based Rice Classification System** — was developed in collaboration with Alkaram Rice Engineering (PVT) Ltd. and deployed on Namal's HPC Cloud, classifying 7 Pakistani rice varieties using handcrafted features and XGBoost.

## Technical Background

My programming experience spans **Python, C++, C, Verilog, and MATLAB**, with hands-on work in both software and hardware domains. On the ML side, I work with TensorFlow, PyTorch, scikit-learn, OpenCV, and Streamlit. On the engineering side, I've designed and simulated circuits in Proteus, built pipelined processors in Verilog, implemented PID controllers on LEGO robots using Simulink, designed patch antennas in Ansys HFSS, and configured campus-scale networks in Cisco Packet Tracer.

I'm comfortable moving between abstraction layers — from writing low-level embedded code on ATmega328P to training neural networks for image classification to deploying web applications on cloud infrastructure.

## Current Work

My GitHub repositories represent a portfolio of **13+ university and research projects**, ranging from a 32-bit pipelined RISC-V processor and a PID-controlled robot to ML-based image classifiers and network subnetting designs. These projects were developed individually and in teams across courses in computer architecture, control systems, machine learning, digital image processing, wireless communication, and computer networks.

I'm currently exploring **real-time ML inference on embedded devices**, **brain and biosignal decoding for BCI applications**, and **Cloud–Edge hybrid AI architectures** — work that sits right at the boundary of what I studied in engineering and where I see the future of intelligent systems heading.

## Contributions

I believe good engineering is built on collaboration. If any of my repositories are useful to you — whether for learning, reference, or building on top of — feel free to explore and contribute. Here's how:

```
1. Fork the Repository: Click "Fork" at the top of any repo to create your own copy.
2. Create a New Branch: Make changes in a descriptive branch (e.g., feature-addition or bugfix-issue).
3. Submit a Pull Request: Once ready, submit a PR with a clear description of your changes.
4. Review and Merge: I'll review it, and if everything checks out, I'll merge it in.
```

Whether it's fixing a bug, improving documentation, or proposing a new feature — all contributions are welcome. Let's build something meaningful together.
